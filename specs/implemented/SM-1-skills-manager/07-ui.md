## Обзор

UI приложения на Svelte 5 (runes) + Skeleton UI v4 + Tailwind CSS 4 в renderer-процессе. Экраны: App Shell с навигацией, Источники, единый Каталог, детали skill, Настройки, центр Уведомлений. UI потребляет IPC-контракты частей 2–6 через `window.api` и не содержит бизнес-логики — только представление, состояние экранов и вызовы IPC. Длительные операции отображаются с прогрессом.

## Пользовательская история

Как пользователь, я хочу удобный интерфейс, где я управляю источниками, просматриваю и фильтрую единый каталог, устанавливаю и обновляю skills, вижу уведомления и настраиваю поведение приложения, чтобы вся работа со skills была в одном месте.

## Поведение

### Затрагиваемые компоненты / экраны

- **`AppShell`** — навигация (боковая/табы), индикатор непрочитанных уведомлений, глобальные тосты.
- **`SourcesPage`** — список источников, `AddSourceDialog` (форма по типу), действия per-source, статус/ошибки.
- **`CatalogPage`** — `SearchBar` (текст), `SourceFilter`, `StatusFilter`, `SortMenu`, виртуализированный список `SkillCard`, пагинация.
- **`SkillDetail`** — панель/страница деталей + действия.
- **`SettingsPage`** — разделы: Обновления, Установка, Git/Сеть, Источники.
- **`NotificationsPage`** — список `NotificationItem`, отметка/очистка.
- **State (Svelte 5 runes):** сторы экранов на `$state`/`$derived`; подписки на IPC-события (`job:*`, `catalog:updated`, `update:checked`, `notification:new`).

### Навигация

- Стартовый экран — Каталог (или онбординг при отсутствии источников: предложить добавить первый источник, как first-launch в reference).
- Из карточки skill → детали (панель/маршрут); назад — к каталогу с сохранением фильтров.
- Из уведомления `update_available` → переход к соответствующему skill в каталоге.

### UI-паттерны прогресса

- Установка/обновление/индексация: карточка операции со статусом `pending → running → ok/skipped/failed`, разворачиваемый лог (по образцу reference `RunCard`/`ExecutionPanel`), авто-следование за активной операцией.
- Тосты для коротких событий; центр уведомлений — история.

### Настройки (маппинг на ConfigStore, Часть 1/6)

- **Обновления:** `checkOnLaunch`, `scheduleEnabled` + интервал/cron, `watchLocalSources`, `autoInstallUpdates`.
- **Установка:** директория установки, целевые агенты (мультивыбор), scope (global/project). При изменении набора агентов — подтверждение + запуск реконсиляции симлинков установленных skills (КП12).
- **CLI:** путь к бинарю `skills` (для офлайн/предустановленного CLI) и npm-registry (корпоративное зеркало) — оба необязательны (эпик Q-03).
- **Git/Сеть:** режим Git (SSH/HTTPS), прокси; секреты вводятся через защищённое хранилище (эпик Q-04), не отображаются в открытом виде.
- **Приложение:** проверка обновлений приложения (electron-updater), кнопка «Проверить сейчас», флаг авто-загрузки (эпик Q-06).
- **Источники:** дублирует управление с `SourcesPage` (единый компонент).

## Требования к данным и API

UI использует IPC-контракты, определённые в частях 2–6 (`source:*`, `catalog:*`, `install:*`, `update:*`, `notifications:*`, события `job:*`). Собственных доменных моделей UI не вводит — использует общие из `src/shared/domain` (Часть 1). ViewModel-обёртки допустимы локально для представления.

## Пограничные случаи и обработка ошибок

- **Нет источников:** онбординг-экран с CTA «Добавить источник».
- **Пустой каталог / нет результатов поиска:** empty-state с подсказкой.
- **Источник в ошибке:** визуальный маркер + подсказка (проверить URL/auth/путь), кнопка «Повторить».
- **Длинные названия/описания:** обрезка с тултипом.
- **Ошибка установки/обновления:** inline-статус в карточке операции + уведомление; кнопка «Повторить».
- **Отсутствие прав/недоступность нативного диалога выбора каталога:** понятная ошибка.
- **Большой каталог:** виртуализация; дебаунс поиска (~300 мс, как в reference).

## Заметки по реализации

**Расположение:** `src/renderer/src/` — Svelte 5 (runes) + Skeleton UI v4 + Tailwind 4.

**Структура:** `lib/stores/*.svelte.ts` (runes-сторы: `ui`, `config`, `catalog`, `sources`, `notifications`, `jobs`), `lib/labels.ts`, `lib/api.ts` (типизированный `window.api`), `components/` (`CatalogPage`, `SkillDetail`, `SourcesPage`, `AddSourceForm`, `SettingsPage`, `NotificationsPage`, `JobsBar`), `App.svelte` (shell: боковая навигация + роутинг по `ui.view` + баннер обновления приложения + `JobsBar`).

**Ключевые решения:**
- **Состояние на runes** (`$state`/`$derived`) в классах `*.svelte.ts`; подписки на IPC-события (`onCatalogUpdated`, `onSourceIndexed`, `onNotification`, `job:*`, `onAppUpdateStatus`) переустанавливают состояние реактивно.
- **UI без бизнес-логики** — только вызовы `window.api` и представление; типобезопасность сквозная через общий `IpcApi`.
- **Каталог:** поиск (дебаунс не добавлен — запрос на каждый ввод, индекс in-memory быстрый), фильтр по статусу, сортировка, пагинация, карточки со статус-бейджами, действия установить/обновить, панель деталей.
- **Настройки:** мультивыбор агентов с **немедленной реконсиляцией симлинков** (`install:reconcileAgents`, КП12), scope, расписание обновлений (`update:setSettings`), CLI-путь/registry/прокси, флоу обновления приложения.
- **Прогресс** активных операций — `JobsBar` (нижняя панель) с отменой; индикатор непрочитанных уведомлений в навигации.

**Осознанные упрощения (deferred, не блокируют функциональность):**
- Автоматические renderer-тесты (jsdom + Testing Library) не заведены — вместо этого сквозная проверка типов (`svelte-check` 0 ошибок), сборка и smoke-boot; ручной чек-лист ниже.
- Нативный диалог выбора каталога для локального источника → пока текстовый ввод пути (диалог — follow-up через IPC `dialog`).
- Виртуализация списка → пагинация (для сотен записей достаточно; виртуализацию добавить при росте).
- Подтверждение-превью перед реконсиляцией агентов и разворачиваемые логи операций → follow-up.
- Переключатель тёмной/светлой темы → тема Skeleton `cerberus` фиксирована; тумблер — follow-up.

**Верификация:** `svelte-check` — 0 ошибок (110 файлов); prettier чисто; `npm test` — 42/42 (бэкенд); `npm run build` — renderer-бандл собран; smoke-boot полного приложения (UI монтируется, IPC-вызовы `config.get`/`catalog.query`/`source.list`/`notifications.list` на старте) — без ошибок.

**Follow-up задачи:** нативный диалог выбора каталога; виртуализация каталога; renderer-тесты; тема-тумблер; превью-подтверждение реконсиляции + разворачиваемые логи задач.
