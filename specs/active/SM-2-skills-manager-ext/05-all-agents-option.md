## Обзор

Добавить в выбор целевых агентов опцию «Все доступные» — быстрый выбор всех агентов. Соответствует семантике `--agent '*'` (часть `--all`) в CLI `skills`.

## Пользовательская история

Как пользователь, я хочу одним переключателем выбрать «все доступные агенты» при установке, чтобы не отмечать их по одному.

## Критерии приемки

- [ ] КП1: В блоке выбора агентов (Настройки → Целевые агенты; и/или контекст установки) есть опция «Все доступные».
- [ ] КП2: Выбор «Все доступные» приводит к установке во все агенты согласно выбранной семантике (см. Q-01).
- [ ] КП3: Опция согласована с реконсиляцией симлинков SM-1 (смена набора агентов применяется корректно).
- [ ] КП4: Визуально видно, что активен режим «все» (а не перечень галочек).

## Поведение

### Затрагиваемые компоненты

- **`src/renderer/.../SettingsPage.svelte`** — переключатель «Все доступные» в блоке целевых агентов.
- **`src/main/installer/exec.ts`** — при режиме «все» для official-установки передаётся `-a '*'` либо разворачивается в список `cliFlag` всех известных агентов (зависит от Q-01).
- **Домен агентов / установочный запрос** — способ представления «все агенты» (спец-значение или разворот в `KNOWN_AGENTS`).

### Контракты

- Семантика «Все доступные» — **гибрид** (решение Q-01):
  - **official** → передавать CLI `-a '*'` (все агенты, которые знает CLI);
  - **git/local** → разворачивать в наши `KNOWN_AGENTS` (звёздочки в файловой установке нет).
  - Ограничение: для official через `*` установки в агентов вне `KNOWN_AGENTS` не отслеживаются нашим сканером/реконсиляцией — задокументировать (детект неполный для нетрекаемых агентов).

## Пограничные случаи и обработка ошибок

1. **Смена «все» → конкретные** — корректная реконсиляция (снятие лишних симлинков).
2. **Официальный vs git/local** — для git/local «все» всегда разворачивается в `KNOWN_AGENTS` (нет CLI-звёздочки); согласовать с Q-01 для official.

## Зависимости и риски

- Взаимодействует с реконсиляцией агентов SM-1.
- При выборе буквального `*` (Вариант B) — расхождение между «установлено CLI» и «видно нашему сканеру» (см. риск детекта в SM-1).

## Заметки по тестированию

- Ручной: включить «Все доступные», установить official и git-скил, проверить реконсиляцию и отображение установок.

## Открытые вопросы

## Q-01: Семантика «Все доступные»

**Приоритет:** Блокирующее
**Вопрос:** «Все доступные» — это все наши известные агенты (`KNOWN_AGENTS`) или буквальный `-a '*'` CLI (все агенты CLI, включая неотслеживаемые)?
**Варианты:**
1. Развернуть в `KNOWN_AGENTS` — согласовано с трекингом/сканированием/реконсиляцией (рекомендуется).
2. Буквальный `-a '*'` для official — максимальное покрытие, но неполный детект/реконсиляция наших установок.
3. Гибрид: official → `-a '*'`, git/local → `KNOWN_AGENTS`.
**Допущение:** Вариант 1.
**Статус:** Отвечен
**Ответ:** Вариант 3 (гибрид): official → `-a '*'`; git/local → разворот в `KNOWN_AGENTS`. Примечание: для official-установок через `*` детект/реконсиляция наших установок будет неполной для агентов вне `KNOWN_AGENTS` — задокументировать в реализации.
