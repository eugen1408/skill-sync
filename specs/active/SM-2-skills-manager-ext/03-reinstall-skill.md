## Обзор

Добавить действие «Переустановить» для установленного скила — принудительная переустановка (force) в те же целевые агенты. Переиспользует `InstallerService` из SM-1 (установка с `force = true`).

## Пользовательская история

Как пользователь, я хочу переустановить установленный скил (например при повреждении файлов или для отката к текущей версии источника), чтобы восстановить корректное состояние без ручного удаления.

## Критерии приемки

- [ ] КП1: В деталях установленного скила (`SkillDetail`) есть кнопка «Переустановить».
- [ ] КП2: Действие запускает установку с `force = true` для агентов, где скил уже установлен (см. Q-01).
- [ ] КП3: Результат/ошибки отображаются как для обычной установки (job-события, тосты, уведомления SM-1).
- [ ] КП4: Для official-скила с предупреждениями безопасности показывается то же подтверждение аудита, что и при установке (SM-1 `installWithAuditGuard`).

## Поведение

### Затрагиваемые компоненты

- **`src/renderer/.../SkillDetail.svelte`** — кнопка «Переустановить» для установленных скилов; вызывает установку с `force = true`.
- **`src/renderer/src/lib/install.ts`** — переиспользовать/расширить хелпер установки для режима переустановки (force + аудит-guard).
- **`InstallerService`** (SM-1) — изменений не требует: установка с `force` уже поддержана.

### Контракты

- Запрос переустановки = существующий `InstallRequest` с `force = true` и `targetAgents` = агенты текущих установок скила (`entry.installations[*].agent`); если пусто — целевые агенты по умолчанию из настроек.

### Навигация

- Диплинки — не требуются.

## Пограничные случаи и обработка ошибок

1. **Скил не установлен** — кнопка «Переустановить» не показывается (доступна только для установленных).
2. **Установка в процессе** — учесть параллелизм SM-1 (job-и); повторный клик не должен запускать дубликат для того же скила (по возможности).
3. **Ошибка переустановки** — уведомление об ошибке (как в SM-1).

## Заметки по тестированию

- Ручной: переустановка official/git/local-скила; проверка, что затронуты только уже установленные агенты.

## Открытые вопросы

## Q-01: Цель переустановки — текущие агенты или дефолтный набор

**Приоритет:** Высокое
**Вопрос:** Переустанавливать в агенты, где скил уже установлен, или в текущий набор целевых агентов из настроек?
**Варианты:**
1. В агенты текущих установок скила (рекомендуется — «восстановить как есть»).
2. В текущий дефолтный набор целевых агентов.
**Допущение:** Вариант 1; если установок нет — дефолтный набор.
**Статус:** Отвечен
**Ответ:** Вариант 1 — в агенты текущих установок скила; если установок нет — дефолтный набор целевых агентов.
